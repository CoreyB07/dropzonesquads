# DropZoneSquads Work Plan + Product Checklist

_Date: 2026-03-25_

## Core Goal

Turn DropZoneSquads from a functional project into a clear, polished, intentional product.

This means not just fixing bugs or restyling isolated pages, but making sure the **whole website makes sense**:
- every section earns its space
- every page supports the product goal
- mobile feels good
- admin tools are useful
- features feel connected
- stale or leftover UI gets removed

---

# Phase 0 — Whole-Site Sense-Making Audit

## Goal
Go through the entire website and verify that everything makes sense.

For every page, section, block, stat, button, and label, ask:
- Does this need to exist?
- Is it useful?
- Is it clear?
- Does it match the product direction?
- Is it redundant?
- Is it helping the user move forward?
- Does it still make sense on mobile?

## Audit lens
Each item should be tagged as one of:
- **Keep**
- **Simplify**
- **Merge**
- **Move**
- **Remove**
- **Upgrade**

## Cross-site checklist
- [ ] Review homepage sections for anything stale, redundant, or unclear
- [ ] Review whether the active filters block/stat actually helps users
- [ ] Review whether any leftover featured-picks language or concepts still exist
- [ ] Review all homepage stats blocks for usefulness
- [ ] Review onboarding / quick-start UI for necessity and size
- [ ] Review squad cards for useful vs unnecessary metadata
- [ ] Review messaging for anything that feels confusing, too heavy, or out of place
- [ ] Review profile for sections that feel too dashboard-like or fragmented
- [ ] Review admin dashboard wording, grouping, and moderation flow clarity
- [ ] Review nav labels and CTA wording site-wide
- [ ] Review empty states and loading states across pages
- [ ] Review mobile layout rhythm across key pages
- [ ] Identify leftover legacy ideas or partial redesign remnants

# Product Standard Going Forward

Every change should make the site more:
- useful
- understandable
- lighter
- more intentional
- more mobile-friendly
- more trustworthy
- more fun to use

If a section does not clearly help the product, it should be simplified or removed.
