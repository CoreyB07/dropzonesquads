# DropZoneSquads UX Audit + Redesign Plan

_Date: 2026-03-25_

## Goal

Make DropZoneSquads feel smoother, lighter, more mobile-friendly, and more product-focused without losing the dark tactical identity.

This plan is meant to guide redesign work in small, shippable passes.

---

## Core Product Direction

DropZoneSquads should feel less like a traditional marketing homepage and more like a **mobile-first squad matchmaking app**.

The experience should optimize for:
- fast understanding
- quick browsing
- easy scanning
- simple actions
- low friction
- better mobile flow

---

## Top-Level Findings

### 1. Homepage flow needs tightening
The homepage currently works, but it still feels like multiple experiences stacked together:
- hero / landing page
- onboarding surface
- app dashboard
- listings browser

The result is that the user has to scroll through too much structure before the browsing experience fully takes over.

### 2. Mobile hero is too tall
The hero section currently takes up too much vertical space on mobile. It looks dramatic, but it delays access to the real product: squads.

### 3. Squad cards are a little too large
The cards are readable, but they are not as scan-friendly as they could be. They use generous spacing, multiple pills, and a relatively tall action area.

### 4. Messaging is functional but still feels heavy
The messaging system is usable, but the current presentation feels more like a panel/dashboard than a polished chat product.

### 5. Profile page is useful but dense
The profile area contains a lot of good information, but it could be organized more clearly and feel more personal, less like a control center.

### 6. Visual system needs one more cleanup pass
The dark tactical look works, but some areas still feel over-styled:
- too many boxes stacked on boxes
- heavy borders
- too much uppercase competition
- too many decorative treatments in some views

---

## Research Summary

General UX guidance for hero sections and marketplace/community apps points toward these patterns:
- hero sections should explain the product and move users toward action quickly
- mobile-first layouts should reduce vertical bulk and decorative overhead
- marketplace browsing should prioritize search, scanning, and fast decision-making
- onboarding should stay lightweight and not block exploration
- cards should emphasize quick comparison over visual heaviness
- messaging should feel conversational, not administrative

---

## Product-Level Design Direction

Keep:
- dark tactical brand identity
- premium but grounded styling
- strong CTA clarity
- fast squad browsing focus

Reduce:
- excessive height in top-of-page sections
- unnecessary decorative layers
- repeated visual patterns that add weight without adding clarity
- metadata pills when short text descriptions would scan better

Introduce more of:
- compact summaries
- clearer hierarchy
- mobile-first section sizing
- cleaner browsing rhythm
- shorter scan paths

# Redesign Roadmap

## Phase 1 — Homepage Mobile-First Cleanup

### Goals
- reduce hero height on mobile
- tighten spacing above the fold
- reduce homepage vertical bulk
- make browsing feel closer and more immediate

### Checklist
- [ ] Reduce hero min-height on mobile
- [ ] Reduce hero text spacing on mobile
- [ ] Reduce hero padding on mobile
- [ ] Consider hiding or simplifying hero stats on small screens
- [ ] Keep hero to headline + subheadline + two CTAs
- [ ] Review if the “How it works” link earns its space
- [ ] Tighten spacing between homepage sections
- [ ] Confirm homepage feels good before first scroll on mobile

### Notes
The homepage should get users to squads faster. The mobile version should feel compact and intentional, not cinematic for too long.

---

## Phase 2 — Homepage Information Hierarchy Cleanup

### Goals
- clarify what matters most on the homepage
- make the page feel like one product flow instead of stacked modules

### Recommended order
1. Hero
2. Quick actions / lightweight onboarding
3. Filters
4. Featured squads
5. All squads
6. Small trust/value block
7. Footer

### Checklist
- [ ] Review current homepage section order
- [ ] Remove or shrink any section that slows browsing
- [ ] Tighten onboarding presentation further if needed
- [ ] Make sure filters appear early enough to be useful
- [ ] Reduce repeated explanation copy
- [ ] Keep value/trust messaging brief

---

## Phase 3 — Squad Card Redesign

### Goals
- improve scanability
- reduce card height
- make listings easier to compare quickly
- improve mobile density

### Potential design direction
Instead of several pills, try a more compact text-based summary.

Example direction:
- Squad name
- Short descriptor line:
  - `Ranked Resurgence • Chill comms • Crossplay`
- Small secondary row:
  - `3 / 4 players • Open`
- Action row:
  - `View Squad` + `Join` or `Chat`

### Checklist
- [ ] Audit card height on mobile and desktop
- [ ] Reduce outer padding slightly
- [ ] Reduce button row height
- [ ] Test replacing some pills with a short natural-language description
- [ ] Keep only the most important metadata visible by default
- [ ] Make “open / closed / invite only” status clearer and simpler
- [ ] Ensure cards remain visually distinct without feeling bulky
- [ ] Compare old card vs compact card side-by-side

### Notes
Not every attribute needs to become a pill. Descriptive summary text may feel more natural and less cluttered.

---

## Phase 4 — Browse + Filter Experience Polish

### Goals
- make finding squads feel quick and controlled
- reduce visual weight around filter interactions
- improve mobile browsing flow

### Checklist
- [ ] Review filter drawer size and friction on mobile
- [ ] Confirm active filter chips are useful and not too noisy
- [ ] Improve spacing between filters and squad results
- [ ] Make sure browse results start high enough on the page
- [ ] Consider a compact “results summary” row
- [ ] Review whether sorting/status/open slots are visually clear enough

---

## Phase 5 — Messaging UX Polish

### Goals
- make messaging feel modern, conversational, and lighter
- improve mobile list/thread flow
- reduce dashboard heaviness

### Checklist
- [ ] Audit conversation list row density
- [ ] Simplify header styling in message list and thread view
- [ ] Improve active/unread state contrast and clarity
- [ ] Tighten message spacing/grouping if needed
- [ ] Improve empty states and no-message states
- [ ] Test mobile list → thread transition flow
- [ ] Decide whether usernames should remain fully uppercase in messaging UI
- [ ] Review whether profile pictures/avatars would improve social feel later
- [ ] Fix small component inconsistencies discovered during audit

### Notes
Messaging should feel calmer and more app-like. Less control-panel energy, more natural conversation energy.

---

## Phase 6 — Profile Redesign

### Goals
- make the profile page easier to understand
- make it feel more personal and less fragmented
- improve mobile readability

### Checklist
- [ ] Audit all current profile sections
- [ ] Identify what belongs in top summary vs secondary sections
- [ ] Simplify the top profile identity area
- [ ] Group gameplay/account fields more clearly
- [ ] Reduce visual variety between profile cards
- [ ] Make edit actions more obvious and less disruptive
- [ ] Check badge/squad sections for density and readability
- [ ] Improve mobile spacing and section stacking

---

## Phase 7 — Global Visual Cleanup Pass

### Goals
- make the whole product feel more polished and consistent
- preserve brand identity while removing unnecessary weight

### Checklist
- [ ] Reduce unnecessary borders where they don’t help hierarchy
- [ ] Reduce decorative effects that add noise
- [ ] Recheck all-caps usage for readability
- [ ] Tighten spacing rhythm across pages
- [ ] Normalize section and card padding patterns
- [ ] Confirm CTA styles are consistent across key pages
- [ ] Confirm mobile tap targets remain comfortable

---

## Priority Order

### Best order to tackle this work
1. Homepage mobile hero cleanup
2. Homepage flow tightening
3. Squad card redesign / compact card experiment
4. Browse + filters polish
5. Messaging polish
6. Profile redesign
7. Final global cleanup pass

- create a new compact squad card pattern

- redesign messaging to feel more like a native app
- simplify the profile page into fewer, clearer sections

## Working Rule for Redesign

Make changes in small passes.
After each pass:
- test on mobile first
- confirm scanability improved
- confirm nothing critical got harder to find
- commit before moving on
